## Spread-allocation CIBIL rate curve (replaces per-band averaging)

- **Ask:** Stop suggesting rates with the old 50-point band averaging. CIBILâ†’rate should
  follow a non-linear curve: 800+ profiles sit near the lender's best rate, 775-800 barely
  moves, and the penalty steepens sharply below 750. The user specified the shape as a fixed
  allocation of each lender's *own* rate spread: >800 = 3%, 775-800 = 7%, 750-775 = 10%,
  700-750 = 20%, 600-700 = 60% (cumulative knots 0.03/0.10/0.20/0.40/1.00). Final displayed
  rate must floor DOWN to the lower 0.05 (7.19â†’7.15, 7.16â†’7.15, 7.20â†’7.20), computed raw then
  floored.

  settling on 3/7/10/20/60 for >800/775-800/750-775/700-750/600-700, putting the bulk of the
  penalty into the 600-700 band; final-rounding rule was briefly nearest-0.05 then changed to
  floor-to-lower-0.05.)
- **Changed:** `lib/rate-engine.ts` â€” added a `CURVE` knot table + `curveFraction(score)`
  (linear interpolation between knots, score clamped to [600,850], returns a raw fraction).

- **Why:** The old engine averaged within each published 50-point band and floored, which made
  the curve too linear and pushed 800-score profiles well above the lender's true floor. The

  shape on top of each institution's own R_minâ†’R_max spread, so wider-spread lenders spread the

  discarded â€” only the two anchors define the spread. The final raw rate is floored DOWN to the
  lower 0.05 (conservative â€” never quotes a rate below what the curve gives), applied only to
  the final raw value so interpolation error never compounds across segments. The separate

  600-700 slice (40% of the spread) now carries the bottom of the curve directly.
