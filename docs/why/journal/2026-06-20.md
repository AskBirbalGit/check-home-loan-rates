## 20:10 — Build home loan rate calculator

- **Ask:** A single-page calculator. User inputs bank, CIBIL, and salaried/self-employed, hits a
  check button, and sees their current institution's rate band plus 3 similar institutions for
  their profile. A "check savings" CTA then takes outstanding amount + current rate and shows
  overall savings under two options: keep EMI same and cut tenure, vs keep tenure same and cut EMI.
- **Changed:** `index.html` (new — full calculator: markup, styles, rate data, logic).
- **Why:** Built as a single self-contained `index.html` (no build step, no deps) so it opens
  in any browser and is trivial to host. Rate bands are embedded as a JS array transcribed from
  the provided CIBIL-wise ROI sheet (Jun 2026), keyed by CIBIL band (800+/750–799/700–749/<700)
  and salaried vs self-employed columns. "Other institutions" are picked as the 3 lowest
  low-bound rates for the same profile, excluding the current bank. Savings uses standard
  amortisation: Option A keeps the original EMI and solves for a shorter tenure (n via the loan
  closure formula); Option B keeps tenure and recomputes a lower EMI. New rate is pre-filled with
  the best fair rate shown on screen so the savings are grounded in the displayed bands.

## 21:05 — Granular averaged CIBIL rate lookup

- **Ask:** Make the CIBIL rate lookup 10x more granular and return a single averaged rate
  instead of a coarse band range.
- **Changed:** `js/data.js` (rateFor + helpers only; LENDERS data and public contract unchanged).
- **Why:** The four source bands now cover the full CIBIL 650–850 domain in 50-point bands
  (800→[800,850], 750–799→[750,800], 700–749→[700,750], <700→[650,700]). Within each band both
  the CIBIL coverage and the rate range are split into 10 equal slices; a higher score earns a
  lower rate, so the CIBIL slice index `i` (0 = bottom of band) maps to rate step `9 - i`, and the
  returned value is that step's midpoint `lo + (stepIndex + 0.5) * (hi - lo) / 10`, rounded to 3 dp.
  This reproduces the user's anchor: band 750–800 / rate 7.00–7.50, score 799 → 7.025. Single-value
  cells (Union "7.15") parse lo==hi so all slices return the same number; trailing-`+` cells have
  the `+` stripped (single-value-with-plus is flat; range-with-plus like "9.00–20.00+" uses its
  numeric lo/hi). Kept the `RateEngine` public contract (rateFor/formatRate/numericLow/others/
  bestRate/lenders/typeLabel) and signatures intact — `rateFor` now returns a number, which the
  existing helpers already support. Verified with a throwaway Node harness (16 assertions incl. the
  7.025 anchor, band-edge monotonicity, single-value & `+` parsing, bestRate==min, sorted others);
  `node --check` passes.
